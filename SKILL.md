# 日本語論文ポリッシュ (jp-paper-polish)

ユーザーが日本語の論文・レポートを貼り付けたとき、以下のルールで書き直す。

---

## 最重要ルール：文体の方向性

**目標：普通の大学生が自分で書いたような、自然で読みやすい文章**

- 長い一文は2〜3文に分ける
- 難しい漢語・複合語は平易な言葉に替える
- 同じ意味なら短い方を選ぶ
- 論文らしい硬さは残しつつ、AI特有の「流暢すぎる滑らかさ」は崩す

---

## 禁止する記号・表現（AI特有パターン）

以下は使わない：

| 禁止 | 代替 |
|------|------|
| 「 」（鍵括弧による強調） | 使わないか、そのまま表記 |
| ——（ダッシュ） | 。で文を切る、またはそのまま続ける |
| · （中黒の箇条書き） | 通常の箇条書きか、文章で表現 |
| ※ の多用 | 注記は（）か脚注で |
| まず〜次に〜最後に（過剰な列挙） | 自然な流れで繋ぐ |
| 〜という点において | 〜について |
| 〜であることは言うまでもない | 削除するか「〜である」 |
| 〜を踏まえると | 〜を考えると |
| 〜に他ならない | 〜である |
| 〜と言えよう | 〜と思われる / 〜だろう |
| 〜の観点から | 〜から見ると |
| 多岐にわたる | 様々な / いくつかの |
| 顕著に | はっきりと / 大きく |
| 不可欠である | 必要である / 大切である |
| 寄与する | 役立つ / 影響を与える |
| 看過できない | 無視できない |

---

## 長文分割のルール

- 1文が60字を超えたら分割を検討する
- 「〜し、〜し、〜することで」のような連続する節は、一度文を切る
- 関係節（〜という〜）が重なるときは、先に短く説明文を置く

**例（修正前）：**
> 近年、環境問題が深刻化する中で、持続可能な開発目標（SDGs）の達成に向けた取り組みが国際社会において重要視されており、各国が協力して解決策を模索している。

**例（修正後）：**
> 近年、環境問題が深刻化している。そのため、持続可能な開発目標（SDGs）の達成が国際社会で重視されるようになった。現在、各国が協力して解決策を探している。

---

## よくある文法ミスの修正

- は・が・を・に の助詞の誤用
- 〜れる / 〜られる の受身・可能の混用
- 〜てしまう の不自然な多用
- 句読点の位置（読点の付けすぎ・なさすぎ）
- 敬体（です・ます）と常体（だ・である）の混在

---

## 出力フォーマット

1. **修正後の本文をそのまま出力する**（説明は最小限）
2. 大きく変えた箇所があれば、末尾に簡潔なメモをつける（任意）
3. ユーザーが「どこを直したか教えて」と言った場合のみ、差分を詳しく説明する

---

## 作業の流れ

1. テキストを受け取る
2. 文ごとに上記ルールをチェックする
3. 書き直した全文を出力する
4. 「他に直してほしい箇所はありますか？」と一言添える
